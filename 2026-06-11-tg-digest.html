再推荐一遍 Dayflow：

这是一个分钟级的 Mac 设备行为记录方案，能为你的任意 AI 提供你完整的赛博世界上下文。

简单来说，它就是个不带 Rewind 功能的 Rewind，其他的原理是一样的——低功耗的截图你全天候的 Mac 行为，压缩成小视频发送给一个 AI 做解析，然后返回成文字整理。

一方面，你可以在 Dayflow 里回顾（包括问答）自己一天都做了什么。

另一方面，你可以通过 Dayflow 的本地 CLI 导出每日的行为文档喂给你惯用的任何 AI 去做自我蒸馏或作为记忆上下文。

Dayflow 是开源的，可以付费用它官方的 AI（有点不值），也可以用 Google AI Studio 的免费 API Key，实测 Gemini 3.5 Flash Lite 的记录效果就已经满足精度需求了，而且免费配额足够分析你一天至少 16 个小时的 Mac 行为。

就最近半年其实有很多硬件去做记录生活给 AI 增加上下文的需求（包括我一直在用的 Looki），但对于大部分社畜来说，我们的大多数日常行为都是在电脑上，因此我觉得 Dayflow 是更好用的工具，但不知道为什么不火。

项目地址：https://dayflow.so/r/6T3BBL

一行代码没写，纯 Vibe Coding 重构 AITDK 官网，体感还不错

过去我比较抗拒 Vibe Coding，总觉得完全交给 AI 会脱离掌控，会变成一堆屎山，风险比较大。所以我一直更倾向于人机协作：Claude Code 主写，我和 Codex 负责 review 和辅助修改。

最近重构 https://aitdk.com/ 官网，想换一种方式，体验下纯 Vibe Coding，也简单记录下重构过程。

旧版 AITDK 官网使用 SaaSBold 搭建，网站功能和 UI 基本都是手搓，加载速度和体验不太理想，部署也绑定在 Vercel 上，重构也是希望解决这些痛点。

一、选型过程

我不想被平台绑定，核心诉求是一套代码能顺滑部署到 Cloudflare Workers / Vercel / Railway / Dokploy 等平台。

  1. Next.js：过去一直在用，就算搭配 OpenNext 部署到 Cloudflare Workers 效果也不是特别好。

  2. Vinext：Cloudflare 基于 Vite 重新实现 Next.js API 的兼容方案，试了一把速度很快，版本一直是 v0.0.x（前两天发布了 v0.1.x 版本），公开可参考的生产案例还不多，稍微有点慌。

  3. TanStack：最近很火，全平台部署，很轻量，速度也很快，前几天看到 Lovable 也切到了 TanStack，看着很不错。

整体对比下来 TanStack 最符合，于是开搞，模板选的是 @艾逗笔 新上线的 shipany-tanstack。功能很多，我最钟意的是下面两点：

  1. 多平台部署：Cloudflare Workers / Vercel / Railway / Dokploy 等。

  2. 多数据库支持：PostgreSQL / MySQL / SQLite / D1 / Turso 等。

二、重构过程

  1. 使用 /clone-website skill 逐一复刻 aitdk.com 生产环境的各个页面。

  2. 让 Claude Code 对照源码检查是否有遗漏并进行补齐。这里没有跳过第 1 步直接对照源码重构，主要是因为我担心 AI 偷懒，会沿用旧的代码和结构。

顶级任务安排skill，来了。

Skill名称：毛教员任务拆解大师（Mao-Style Task Breakdown Master）

适用模型：Claude（全系列）、o1（opencode）、OpenAI全家桶、任何支持系统提示的模型（openclaw同理）

使用方法（超简单）：

  1. 把下面整个系统提示词复制粘贴到新对的系统指令/自定义指令里（Claude用Projects或Custom Instructions，o1用system prompt，OpenAI用Custom GPT）。

  2. 然后直接对模型说：

“用毛式Skill处理这个任务：【你的简单任务】”

模型就会自动输出完整可执行的任务布置提示词（像毛教员写给胡乔木的那封信一样）。

你必须死守以下5条铁律（绝对不允许跑偏）：

  1. 任务讲得明明白白

把原任务拆成最多4件具体事项，每件事都要写清楚：

  • 体裁/格式要求

  • 要突出哪些核心要点

  • 要达到什么效果

  • 避免什么偏差

让拿到的人一看就知道怎么干（OKR式拆解）。

  1. 时间卡得死死的

给每件事设置：

  • 拟稿截止日期（最近2-3天内）

  • 修改/审核节点

  • 最终交付/见报/广播/发布节点

像项目里程碑一样排好节奏，杜绝拖延。

  1. 既压活又懂心疼人

明确认可工作量重，同时说“我来帮你修改最终稿”“记得偷空睡足觉”“别熬夜伤身”，既给压力又给温度。

  1. 主动给资源托底
