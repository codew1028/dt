如果机器人最终只是完成任务，为什么需要人形？

一、第一天问

半年多前，我转到机器人赛道。

之前做互联网产品，逻辑很顺：理解用户、建立关系、提供服务。但做到某个阶段，撞上一块天花板——

屏幕一关，关系就断了。没有共同空间，没有共享时间线，没有"我知道你在那里"的确定性。

所以看到具身智能这个机会时，很快就做出了决定：物理在场是关系的真正载体。

而行业里的主流叙事是错的。

很多人把机器人理解为Agent的载体，带入了Agent的叙事：主动发现交互机会、预判需求、主动服务。

听起来很美好。

但只要你追问一句：第一天，机器人知道什么？

它不知道你的职业，不知道你的朋友，不知道你的投资人，不知道你最近关注什么，甚至不知道你是谁。

很多所谓的"智能发现"，本质上都是在假设数据已经存在。但创业的第一性问是：第一天你手里有什么？

它站在你家客厅里，能感知到的极其有限。

墙上的海报、书架上的书、柜子里的模型、客厅里的吉他。用户什么时候回家、什么时候睡觉、有没有和它说。

除此之外，可能什么都不知道。

二、在场：唯一不需要数据就能获得的资产

但这些东西恰恰是机器人独有的——它们既不来自互联网，也不来自通讯录，更不来自搜索记录。

它们来自同一个事实：机器人一直在那里。

手机知道互联网，机器人知道家庭，这是两种完全不同的信息。手机知道世界发生了什么，机器人知道你家发生了什么。

三、从观察到共同记忆：真正的护城河

过去一段时间，我在思考Agent叙事里的"主动性场景"——也就是机器人什么时候可以主动开口、主动交互。

后来发现这个问问得太早了。

机器人第一天来到家里，看到墙上挂着篮球海报，柜子里摆着高达模型，书桌上放着一本创业相关的书。

这些都不是机会，甚至不是理解。

只是观察。

但观察有一个特点：它会积累。

某一天机器人发现：那把吉他已经一个月没被拿起来过了。

某一天机器人发现：那个拼到一半的模型已经放了三周。

某一天机器人发现：用户最近每天都在凌晨一点以后睡觉。

用户活在当下。机器人活在时间里。

但记住一件事，不等于拥有共同记忆。

机器人知道"用户有一把吉他"——这只是私有数据。

直到有一天，机器人说：

"你最近好像很久没弹吉他了。"

用户回答：

"哈哈是啊，最近太忙了。"

从这一刻起，那把吉他才变成你们之间的锚点。

关系不是从"我记得"开始的，是从"你也记得"开始的。

互联网产品拥有的是用户数据，而机器人可以拥有共同经历，这是两种完全不同的东西。

四、关系先于建议

很多主动系统的逻辑是：

"观察 → 分析 → 建议"

但现实世界更像：

"观察 → 共同经历 → 关系 → 发言权 → 建议"

同一句："你最近很久没碰吉他了。"

第一天说，像监控。

三个月后说，像关心。

差别不是模型能力是关系。

没有共同经历，机会只是打扰。没有关系，主动性只是噪音。

五、存在设计：被严重低估的工业设计

如果机器人的核心价值来自共同生活，那么工业设计的重要性被行业严重低估了。

过去我们默认：AI是产品，机器人是载体。 所以工业设计更像包装。

但如果机器人的核心资产是在场，事情就变了。

一个软件APP，用户不打开的时候，你不需要设计它的状态——它不存在。

但机器人不一样，它一直在那里。

用户每天可能只和机器人交互几分钟，但感知到它"在"的时间，可能是几小时。

这意味着：工业设计讨论的不是美观，而是用户愿不愿意和这个东西长期共处。

它站在角落充电时是什么姿态？它看到用户回家时会做什么？它不说的时候，用户能不能感觉到它的存在？

这些"不交互时刻"的设计，才是机器人产品真正的差异化。

六、双足的意义：从功能视角到关系视角

过去讨论双足机器人，大家会说：能上楼梯、能适应人类环境、通用性更强。

这些都成立，但这是功能视角。

如果从在场的视角看，问会变成：

什么样的存在形式，更容易进入人类的生活？

轮式机器人很容易让人联想到工具：扫地机器人、配送机器人、行李箱。

双足机器人让人联想到另一套东西：人、宠物、玩具、角色。

这里讨论的甚至不是能力，而是关系预设。

用户最终建立关系的对象，往往不是功能体，而是角色。

人类历史上最成功的陪伴产品——宠物、玩具、IP角色——很多时候并不强大，甚至没什么功能。但人们愿意和它们长期相处。

双足的姿态变化，在"存在"的语境下反而是优势。猫跳上桌子会调整重心，狗奔跑时会变换姿态。这种"动态平衡"传递出"这是一个需要保持姿态的活物"，而不是"一个运行中的机器"。

七、竞争转移

我越来越相信，机器人行业未来最大的竞争，未必只是模型的竞争。

而是："智能设计 + 存在设计"的竞争。

智能正在变得越来越便宜。

但一个长期生活在用户身边、与用户共享同一段时间线、拥有共同记忆和共同经历的存在会是稀缺的。

八、重新定义链路

最开始，我把机器人理解成Agent。

关注：Goal → State → Planning → Opportunity → Execution。

后来慢慢发现，真正的链路应该是：Presence → Observation → Shared Memory → Relationship → Influence → Opportunity

机会和主动性当然重要，但它们只是长期共同生活之后，自然长出来的一种能力。

九、回到开头的问：如果机器人最终只是完成任务，为什么需要人形？

不是因为人形更能干活。而是因为"人形更容易被允许留在那里。"

智能设计解决的是"能做什么"，存在设计解决的是"能被允许存在多久"。

后者，才是机器人行业的护城河。

推荐这篇文章，作者完整展示了怎么用 LLM 做一个真正好用的个人工具，重要的是，他展示了如何让 LLM 取代那些你之前觉得不好用的产品，服务，非常值得参考。

作者分享了自己的真实经验，用了个月、迭代了多个版本的产品。作者遇到的每一个坑（模型选错、prompt 架构设计、延迟优化）都是实战教训。如果你也想用 LLM 改造自己的工作流，这篇是最佳案例。

我取消了 $200/月的法语家教，自己写了个 LLM 替代品，效果更好

几个月前，我还在用传统方式学法语——每周一次 45 分钟私教课，每月 $200。导师学术背景扎实，教学有方。但几个月下来，我发现讲了很多，记住的却不多。

每节课前 20 分钟在对，然后花大量时间回顾已经讲过的内容。新语法点进展很慢，我越来越不耐烦。

我读过足够多关于遗忘曲线的文章，也知道 LLM 工具摆弄了挺久。所以开始想：能不能自己做一个——更好，更便宜？

问不是教学质量。是两个结构性问。

第一，复习节奏。人类导师按日历来排课。我需要的算法应该按遗忘曲线来排——在刚要遗忘之前正好复习，而不是在随机时间迟来的回顾。这叫间隔重复，背后有 40 年的记忆研究。Anki 这类 APP 已经证明了它有效。问是怎么把它应用到比闪卡更细腻的东西上。

第二，错误追踪。我的导师大概记得我哪里弱，会在合适的时候测试我。但一个类型为"Alex 在虚拟语气上挣扎"的数据点不够用。我需要的是："Alex 在否定句中持续用 avoir 做 aller 的 passé composé 助动词。"这才够精确，解决起来才快。

我还琢磨：真的要花钱就为了对前 20 分钟，而且一周只能聊一次？我能不能随时练口语，免费或者几乎免费？

所有这些，感觉都是可以解决的软件问。

架构：两个关键组件

一个知识库导师，一个语音练习应用，互相配合。

知识库导师是一个基于 Claude 的文本工作流。它维护一个 JSON 文件，追踪我学过的每一个语法点——上一次什么时候复习、当时信心如何、具体哪里错了、什么时候又该复习了。每次 session 前，它读这个文件，找出该复习的内容，生成难度递增的练习：先选择，再填空，再翻译，最后自由输出。session 结束后，所有数据写回文件。

底层算法是 SM-2——Anki 用的同一个。每次练完一个 topic，我给自己打分 1 到 4。分数调整两个参数：这个 topic 对我来说的自然掌握程度，和下次复习的时间间隔。连续四次满分，三周后见。彻底忘了，明天见。久而久之，它构建出一个按 topic 划分的、针对我个人的遗忘曲线模型。

弱点追踪是真正有用的地方。不记录"Alex 在 passé composé 上挣扎"，而是记录"否定句中运动动词用错助动词——自 2026-04-12 起"。下次这个 topic 被复习时，练习就围绕这个具体结构来出——不是泛泛的 passé composé 练习，而是精确炸那个曾经过不去的点。

语音导师叫 Causons，是一个浏览器应用。点一下麦克风，说，听到同一个语言的回复。启动时它读取同一个知识库文件，把对引导向该复习的 topic 和已知的薄弱点。它还会在对流中纠正语法错误——但只纠正那些客观上就是错的、母语者绝对不会说的东西。转录错误、标点、语域选择、口语缩略——默认忽略。

两个组件分工明确。知识库导师建立知识。Causons 在对场景中激活它。你可以理论上掌握反身代词的规则，然后在真要说出口的时候完全卡住。语音应用的作用就是打破这个壁垒。

语音管线是三次 API 调用串联：

语音转文字走 Groq，跑 Whisper-large-v3。Azure 认知服务有免费层，看起来是低成本首选。没用多久就推送到付费层——不是因为用量超标，就是开始用免费层了。体验很差。换到 Groq，同一个模型，硬件更快（200-500ms vs OpenAI 的 800-1500ms），价格约 $0.002/分钟。

对走 gpt-4o-mini。这块费了挺长时间。最初用 Groq 的 Llama 3.3 70B，因为更便宜、能力不差。日常规范法语还行，但碰到完全正常的口语表达会自信地解释为什么它错了——实际没错。花了很长时间 prompt engineering，最后结论是问不在 prompt，在前端模型对口语化法语语法的底层知识不行。切到 gpt-4o-mini 解决了。每 session 成本差异可以忽略。

文本转语音走 OpenAI tts-1，和 gpt-4o-mini 共用一把 key。Nova 和 Onyx 音色自动适配语言，无需配置。ElevenLabs 音质更好，但第三家 provider 和 $5/月起步价不值得。

总结：一次 15 分钟的 session，三道 API 总共约 4 美分。

Prompt 架构的设计思路也很有启发。

语言导师的 prompt 里有两种性质完全不同的内容。教学策略——什么时候该纠正、怎么措辞、什么语域——对每个用户基本稳定，应该跨 session 保持不变。上下文数据——今天练哪个语言、哪些 topic 该复习了、最近错了哪些东西——每次都在变，应该每次都从源重建。

第一版把两者混在一起，一个星期后 prompt 里塞着过期的 topic 列表。修复方法是把它们当两类数据来对待：稳定的教学策略存在 IndexedDB 里直接用，数据驱动的部分从真实的源（语言下拉框、知识库文件、今天日期）每次重建。还有一个组装顺序：稳定的放前面（LLM provider 可以缓存 prefix），会变的部分放后面。

还有一个明确的原则：假纠正比漏纠正更糟。 如果导师把正确的东西标错，你知道的情况下会惹恼你，你不知道的情况下是学习损害。

延迟优化方面：从 2-6 秒砍到 0.8-2.5 秒。靠的是 MediaSource Extensions——不等整个 MP3 下载完，边接收边播。再加上不等完整句子，看到 25 个字符以上就发第一段 TTS 请求。

最后：安保可以不用管。在个人设备上本地存储 key 是 fine 的，有 PIN-based 加密的草图但优先级不高。

回头看两个教训。在错误的模型上待太久了。 一旦 Llama 开始幻觉语法纠正，我应该马上切。Prompt 里不要把稳定策略和变化数据混在一起——分开处理。

两个组件都在 GitHub 上。知识库导师用 Claude 项目管理起来就行。Causons 一行 npm install，一行 deploy 到 Cloudflare Pages。

# 公众号被限流后，我是怎么慢慢恢复的

5 月份，我的公众号突然开始不对劲。

不是某一篇数据差，而是连续很多篇文章，阅读量都只有个位数。刚开始我没反应过来，还以为是自己内容出了问题，选题不行，表达也不行，所以那段时间我还在继续发。

直到有一天，我顺手去做了账号检测，才发现真正的问题不是内容，而是账号被“限制推荐”了。

看到那几个字的时候，我一下就明白了。不是文章没人看，而是文章根本没有被正常推荐出去。那种感觉挺复杂的，一方面有点慌，另一方面又有点松口气。至少，我终于知道问题出在哪了。

知道账号被限制推荐之后，我没有继续硬发，而是开始回头排查以前的文章。

后来我在 X 上看到一篇文章，专门讲微信限制推荐的处理方法。里面提到，从 2026 年 5 月开始，微信对导流、引流类内容查得特别严。哪怕你导的是自己的微信，只要文章里出现二维码、微信号，或者明显引导用户加你、找你的内容，都可能被判定有问题。

我看完之后，基本就对上了。

因为我以前确实在文章里放过二维码，也写过一些带引导意味的话。当时觉得很正常，现在回头看，确实容易踩线。

找到原因之后，我就开始处理旧文章。

第一步，是把明显违规的内容全部删掉。包括二维码、微信号，以及所有带导流意味的表达。

第二步，是把最近几个月发过的文章重新翻一遍。不是只改一两篇，而是尽量把今年以来的内容都过一遍。只要可能有问题，我就处理掉。检测出来违规比较明显的文章，我直接删除。

这个过程挺麻烦，因为你要一篇篇去找，一篇篇去改。但没办法，既然知道问题出在这里，就只能老老实实处理。

处理完之后，我开始继续发内容。

只是这次我很克制，不再碰任何导流的东西，只发原创，也尽量让内容干净一点。那段时间其实挺难熬，因为数据还是不好，发出去照样是个位数阅读。你会怀疑，到底有没有用，也会怀疑这个号是不是已经很难恢复了。

但我还是坚持发了下去。

大概坚持了两周，昨天我又去检测了一次账号，结果终于变了。

账号状态恢复正常了。

虽然还没有拿到推荐流量，但至少“限制推荐”这个问题已经解除了。对我来说，这说明前面做的那些事是有效的。

如果你也遇到类似情况，我觉得可以先做这几步。

先去检测账号状态，别一上来就怀疑是内容问题。

如果真的是限制推荐，就先排查旧文章，重点看有没有二维码、微信号、导流话术这些内容。

然后把最近几个月的文章一起清一遍，能改就改，明显违规的就删。

最后，继续稳定发原创内容，别着急，也别再重复踩同样的坑。

我现在还在继续更新。

虽然推荐流量还没完全回来，但至少账号已经恢复正常。后面能不能慢慢起来，我不知道，但起码方向是对的。
